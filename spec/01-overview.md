# 1. Обзор проекта

## Назначение

Бэкенд для Telegram-бота, который обрабатывает изображения с помощью нейросетей (Replicate). Архитектура — полностью бессерверная (Serverless) на базе Yandex Cloud.

## Ключевая идея: S3-as-DB

Отказ от баз данных (SQL/NoSQL) в пользу **Object Storage (S3)** для хранения контекста задач (state management). Состояние между запросом пользователя и вебхуком от Replicate хранится в JSON-файлах в S3 по пути `tasks/{prediction_id}.json`.

## Цели

- Обработка фото пользователя через Replicate.
- Связка «пользователь → Telegram → наш бэкенд → Replicate → вебхук → ответ в Telegram» без использования БД.
